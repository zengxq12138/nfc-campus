# 一期 PRD：广西大学 NFC 毕业留念微站

## 1. 项目概述

- 项目目标：制作一个轻量、精美的 Web 留念应用，配合嵌入小礼物中的 NFC 标签使用。每个 NFC 标签写入带 `id` 查询参数的访问地址，访客触碰礼物后进入对应毕业生的留念页面。
- 要解决的问题：毕业礼物本身承载的信息有限，本项目用 NFC 入口把个人毕业信息、照片、签名和广西大学风貌连接起来，形成可长期保存、可编辑的数字留念页。
- 一期要验证的核心价值：毕业生和同学通过一件小礼物即可打开专属留念页，既能看到个人毕业信息，也能回看母校风貌，整体体验需要比普通表单页更有纪念感和设计品质。

## 2. 目标用户

- 主要用户：毕业生本人、同班同学、收到 NFC 留念礼物的人。
- 次要相关方：班级组织者、项目维护者、后续负责录入 NFC 标签的人。
- 关键使用场景：
  - 班级为每位毕业生准备一个小礼物，礼物内嵌 NFC 标签。
  - NFC 标签写入形如 `https://domain.com/?id=student001` 的访问地址。
  - 访客触碰礼物后打开页面，看到广西大学风貌介绍和对应毕业生资料。
  - 毕业生首次进入个人信息界面，如果未设置密码，需要先设置密码。
  - 毕业生后续编辑资料时，在确认保存阶段输入密码完成校验。

## 3. 需求边界

### 3.1 一期纳入范围

- 广西大学风貌展示：
  - 学校基本概况。
  - 4 到 6 个标志性建筑或地点的图文展示。
  - 图片资源优先来自广西大学官网等公开互联网来源。
  - 内容以“母校记忆”和“毕业留念”为语境，不做招生官网式信息堆叠。
- 毕业生个人信息展示：
  - 姓名。
  - 专业班级。
  - 毕业照或个人照片，最多 10 张，以轮播图形式展示。
  - 个性签名。
- 毕业生个人信息编辑：
  - 根据 `?id=xx` 识别当前毕业生记录。
  - 首次访问个人信息界面，如未设置密码，先设置密码。
  - 修改个人信息时，保存确认阶段校验密码。
  - 支持更新文字信息和照片。
- 数据设计：
  - 一期按“一张毕业生个人信息表”设计。
  - 数据量预计较小。
  - 字段可能频繁变化，因此字段设计需保留一定扩展空间，但不做复杂动态表单系统。
- 前端设计：
  - 前端是本项目最重要部分。
  - 页面应精美、图文并茂、移动端体验优先。
  - 设计方向按纪念型校园微站处理：校园影像、留念感、清爽但不模板化。

### 3.2 一期明确不做

- 不做复杂账号体系、注册登录、短信验证码、微信授权登录。
- 不做管理员后台和细粒度权限管理。
- 不做班级通讯录、评论墙、点赞、留言审核等社区功能。
- 不做支付、订单、物流等礼物生产流程。
- 不做复杂数据统计和访问分析。
- 不做完全动态字段配置系统，一期只预留扩展字段或 JSON 字段。
- 不做多学校、多班级、多租户能力。
- 不在需求阶段直接编码。本阶段先完成 PRD，技术方案和前端设计稿进入第二阶段后再输出。

## 4. 核心业务流程

1. NFC 标签准备：
   - 为每个毕业生分配唯一 `id`。
   - 生成带 `id` 的访问地址。
   - 将地址写入 NFC 标签。
   - 将 NFC 标签嵌入毕业留念小礼物。

2. 访客查看：
   - 访客触碰小礼物上的 NFC 标签。
   - 浏览器打开带 `?id=xx` 的留念页面。
   - 页面展示广西大学风貌内容。
   - 页面展示该 `id` 对应毕业生的姓名、专业班级、照片轮播和个性签名。

3. 首次设置密码：
   - 毕业生进入个人信息编辑入口。
   - 系统发现该 `id` 尚未设置密码。
   - 毕业生设置密码。
   - 设置成功后可以继续完善资料。

4. 编辑个人信息：
   - 毕业生打开自己的留念页。
   - 进入编辑状态。
   - 修改姓名、专业班级、照片、个性签名。
   - 点击保存。
   - 在确认阶段输入密码。
   - 密码校验通过后保存更新。

## 5. 一期功能清单

### 5.1 必做功能

- 功能项：NFC 参数识别  
  目标 / 说明 / 验收要点：页面能读取 URL 中的 `id` 参数，并据此加载对应毕业生资料；缺少 `id` 或找不到记录时展示明确的空状态。

- 功能项：广西大学风貌页  
  目标 / 说明 / 验收要点：展示学校概况和 4 到 6 个标志性地点；每个地点包含图片、名称和简短介绍；图片资源需注明来源链接。

- 功能项：毕业生资料展示  
  目标 / 说明 / 验收要点：展示姓名、专业班级、照片轮播、个性签名；照片最多 10 张；移动端展示不拥挤、不遮挡文字。

- 功能项：首次密码设置  
  目标 / 说明 / 验收要点：当对应 `id` 未设置密码时，进入编辑流程前要求设置密码；设置成功后可编辑个人资料。

- 功能项：资料编辑与密码确认  
  目标 / 说明 / 验收要点：毕业生可编辑资料；点击保存后必须输入密码；密码正确才写入更新。

- 功能项：基础状态处理  
  目标 / 说明 / 验收要点：提供加载、空记录、密码错误、保存成功、保存失败等状态，不出现无反馈操作。

- 功能项：前端视觉设计规范  
  目标 / 说明 / 验收要点：确定页面信息架构、视觉风格、色彩、字体、图片比例、动效原则和移动端布局规则，为后续编码提供明确依据。

### 5.2 可延后功能

- 管理员后台：一期可通过数据库或脚本录入初始数据，后台会增加开发量。
- 批量 NFC 写入工具：一期可手动生成链接和写入标签，批量工具后续再做。
- 留言墙或祝福墙：会引入内容审核和隐私问题，先不做。
- 公开搜索毕业生：留念礼物场景更适合“知道链接即可访问”，不需要搜索入口。
- 多班级管理：当前目标是一个轻量毕业留念项目，先不做多租户。
- 完全动态字段编辑器：字段频繁变化可通过预留扩展字段解决，先不做复杂配置界面。

## 6. 第一期交付成果

- 交付物 1：一期 PRD 文档，明确目标用户、业务流程、功能边界和验收标准。
- 交付物 2：第二阶段输出概要设计，明确技术栈、数据模型、接口边界和部署方式。
- 交付物 3：第二阶段输出前端设计方案，包含页面结构、视觉方向、关键组件和图片资源策略。
- 交付物 4：后续编码阶段交付可运行 Web 应用，包含展示、编辑、照片轮播、密码设置和密码校验闭环。

## 7. 验收标准

- PRD 验收：
  - 能清楚说明 NFC 留念礼物的核心场景。
  - 明确一期做什么、不做什么。
  - 明确毕业生资料展示和编辑闭环。
  - 明确前端设计是最高优先级。

- 内容验收：
  - 广西大学概况使用权威来源。
  - 校园图片来自公开互联网资源，优先使用广西大学官网。
  - 标志性地点数量控制在 4 到 6 个。

- 产品验收：
  - 访问 `?id=xx` 能定位到指定毕业生。
  - 首次未设密码时能设置密码。
  - 修改资料保存时必须校验密码。
  - 照片最多 10 张，并以轮播形式展示。

- 前端验收：
  - 首屏有明确的广西大学与毕业留念主题。
  - 校园风貌与个人资料之间过渡自然。
  - 移动端优先，NFC 打开后的浏览体验顺畅。
  - 图文比例、排版、动效和表单状态达到“精美”要求，而不是普通后台表单。

## 8. 风险、假设与待确认项

- 风险：
  - 公开图片资源可能存在版权或防盗链限制，需要优先确认可用来源。
  - 个人照片上传涉及隐私和存储安全，需要在技术阶段明确文件大小、格式和访问策略。
  - 用 `?id=xx` 直接访问个人页较轻量，但不等同于强隐私保护。
  - 单表加扩展字段适合小规模快速迭代，但如果后续字段差异很大，可能需要重构为字段配置模型。

- 假设：
  - 一期面向单个班级或小规模毕业生群体。
  - `id` 由项目维护者预先生成，不允许用户自行注册生成。
  - 查看个人留念页不需要密码，编辑才需要密码。
  - 当前阶段先完成文档设计和技术方案，不直接编码。

- 待确认：
  - 是否需要禁止搜索引擎收录个人资料页。
  - 图片上传是由毕业生自己上传，还是由维护者统一上传。
  - 毕业生 `id` 是否需要使用不可猜测的随机串，而不是连续编号。
  - 是否需要统一的班级标题、毕业年份、学院名称和页脚署名。
  - 项目最终部署位置：公网云服务、学校服务器、Vercel/Netlify，或自有云主机。
