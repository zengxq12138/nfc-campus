# 前端设计方案：广西大学 NFC 毕业留念微站

## 1. Design Read

Reading this as: 纪念型校园微站 for 毕业生、同学和收到 NFC 小礼物的人, with a 校园影像 + 青春留念 language, leaning toward Next.js + Tailwind + Motion 的轻量高质感前端。

## 2. 设计目标

- NFC 打开后的第一眼要明确：这是广西大学毕业留念礼物的专属页面。
- 页面要兼顾“母校风貌”和“个人留念”，不能像招生官网，也不能像后台资料表单。
- 移动端优先，因为 NFC 主要由手机触发。
- 图文并茂，真实图片是视觉核心。
- 编辑体验要清楚、轻量、有反馈，但不喧宾夺主。

## 3. 视觉拨盘

- `DESIGN_VARIANCE: 7`  
  使用非模板化布局：首屏影像叠层、校园地点错落卡片、个人资料区域有仪式感，但移动端保持单列稳定。

- `MOTION_INTENSITY: 5`  
  使用轻量动效：首屏淡入、图片切换、卡片进入、编辑面板展开。避免复杂滚动劫持和大面积视差。

- `VISUAL_DENSITY: 4`  
  内容节奏偏舒展。每个模块聚焦一个信息，不把学校概况、地标和个人资料堆在同一屏。

## 4. 页面结构

### 4.1 首屏：NFC 留念入口

- 内容：
  - 广西大学毕业留念标题。
  - 一句简短说明：这是一份来自母校与同窗的毕业纪念。
  - 当前毕业生姓名和专业班级的预览。
  - 一张广西大学校园主视觉图。

- 布局：
  - 桌面端使用非对称 split hero，左侧文字，右侧大图。
  - 移动端首屏单列，图片在上或标题下方，保证姓名和照片可快速看到。
  - 首屏不放过多按钮，最多一个“编辑资料”入口。

### 4.2 广西大学概况

- 内容：
  - 用 2 到 3 段短文介绍广西大学。
  - 只保留与留念相关的信息：历史、南宁、校训、综合性大学、双一流等。

- 布局：
  - 不做长官网文本。
  - 用大标题 + 短段落 + 关键事实块表达。
  - 事实块数量控制在 3 到 4 个。

### 4.3 校园地标图文

- 内容：
  - 4 到 6 个地点。
  - 每个地点包含图片、名称、20 到 40 字介绍、来源链接。

- 布局：
  - 使用 bento / masonry 组合，不做六个等宽卡片。
  - 至少 2 个地点使用大图卡，其他地点使用较小图文卡。
  - 图片不叠加标签，地点名放在图片外，避免模板感。

### 4.4 个人留念区

- 内容：
  - 姓名。
  - 专业班级。
  - 照片轮播，最多 5 张。
  - 个性签名。

- 布局：
  - 个人区是页面的情绪高点，视觉权重不低于校园风貌。
  - 桌面端可使用左侧照片轮播、右侧资料和签名。
  - 移动端照片优先，资料紧随其后。
  - 个性签名用较大字号，但不使用过度诗化文案。

### 4.5 编辑面板

- 内容：
  - 姓名输入。
  - 专业班级输入。
  - 照片上传和排序。
  - 个性签名输入。
  - 首次密码设置。
  - 保存时密码确认。

- 布局：
  - 使用抽屉或独立编辑区，不跳转复杂后台。
  - 表单标签在输入框上方，不用 placeholder 代替标签。
  - 错误信息在字段下方展示。
  - 保存按钮状态包含默认、校验中、成功、失败。

## 5. 颜色与字体

- 主题：light first，支持 dark mode 作为后续增强。编码阶段若时间有限，先保证 light mode 完整。
- 主背景：接近白色或极浅灰，不用米黄色复古纸张风。
- 主文字：深灰或近黑。
- 单一强调色：广西大学相关的深绿色方向，配合校园树木和南宁绿城意象。
- 辅助色：灰阶和低饱和绿色，不使用 AI 紫蓝渐变。
- 字体：优先使用 Geist / Outfit / 系统 sans。中文使用系统黑体栈，确保可读性和加载稳定。

## 6. 图片策略

- 优先来源：
  - 广西大学官网学校概况。
  - 广西大学官网校园景观。
  - 广西大学新闻或官方图集。

- 使用原则：
  - 每张外部图片记录来源 URL。
  - 如外链防盗链，编码阶段下载到项目资源目录或改用可稳定访问的公开资源。
  - 所有图片设置固定比例，避免加载时布局跳动。
  - 每张图片提供准确 alt 文本。

- 推荐图片比例：
  - 首屏主图：`4:3` 或 `16:10`。
  - 地标大图：`16:10`。
  - 地标小图：`4:3`。
  - 个人照片轮播：移动端 `4:5`，桌面端可 `3:4` 或 `1:1`。

## 7. 关键组件

- `HeroSection`：首屏主题、毕业生预览、校园主视觉。
- `CampusOverview`：学校概况和关键事实。
- `CampusSpotGrid`：地标图文 bento。
- `GraduateProfile`：个人资料展示。
- `PhotoCarousel`：最多 5 张照片轮播。
- `EditProfilePanel`：资料编辑入口。
- `PasswordSetupForm`：首次密码设置。
- `PasswordConfirmDialog`：保存前密码确认。
- `ImageSourceList`：图片来源说明。
- `EmptyProfileState`：未找到或尚未完善资料。

## 8. 交互与状态

- 页面加载：
  - 首屏骨架应匹配最终布局，不用居中转圈。

- 找不到 `id`：
  - 展示柔和空状态，说明 NFC 链接未匹配到资料。
  - 不暴露数据库错误。

- 未设置密码：
  - 点击编辑后先进入设置密码流程。
  - 设置完成后继续编辑，不要求重新打开页面。

- 密码错误：
  - 在确认弹窗内提示，不清空已填写表单。

- 保存成功：
  - 更新展示区数据。
  - 显示短暂成功反馈。

- 照片超过 5 张：
  - 上传前阻止，明确提示最多 5 张。

## 9. 移动端规则

- 首屏必须在手机上快速看到主题、姓名和至少一张照片或校园图。
- 所有多列布局在 768px 以下变为单列。
- 按钮高度不小于 44px。
- 编辑面板输入区留足间距，避免键盘弹出后遮挡保存按钮。
- 轮播控件要支持触摸滑动。

## 10. 前端验收清单

- 首屏一眼能看出“广西大学 + 毕业留念 + 当前毕业生”。
- 校园地标使用真实图片，不用纯文字或假插画占位。
- 不出现三列等宽模板卡片堆叠。
- 不使用大面积紫蓝渐变。
- 不使用过度玻璃拟态。
- 不使用复杂后台式表单页面。
- 个人资料展示和编辑状态都有清楚反馈。
- 桌面和手机截图都不出现文字溢出、按钮换行、图片压扁或内容遮挡。
- 图片来源可追踪。
- 页面最终需要用 Playwright 或浏览器截图检查移动端和桌面端。

## 11. 待编码阶段确认

- 最终使用哪些广西大学图片。
- 是否将外部图片下载到项目本地。
- 是否加暗色模式。
- 是否需要单独的分享卡片或 Open Graph 图片。
- 是否隐藏搜索引擎收录个人资料页。
